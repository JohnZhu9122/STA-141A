Wildfire Notable-Event Prediction¶

Abstract¶

This report describes the process of preparing forecasts for next-month wildfire occurrence using monthly data taken from historical fire incident records. The goal is to determine whether at least one notable wildfire will occur in each grid cell during the coming month. To accomplish this, we first process the raw incident data by breaking it down into spatiotemporal units, construct predictors from information available by the end of each month to predict if there is at least one notable wildfire happened in the next month, and analyzing recent fire-history, seasonal, and spatial patterns.

We compare the basic logistic regression model with XGBoost using expanding-window, time-aware validation. Since the outcome is highly imbalanced, model performance is evaluated primarily by the area under the ROC curve (AUC). Overall, the average validation AUC of XGBoost (0.784) is higher than that of logistic regression (0.695), which represents an absolute improvement of 0.089 AUC points.

1. Data Preprocessing¶

This section explains how the raw wildfire incident table is converted into a grid-by-month dataset suitable for prediction.

We cover:

  • How notable fires are identified and filtered
  • How space–time units are constructed (grid × month)
  • How the binary outcome is defined
  • How predictors are constructed (lagged to avoid leakage)
  • How missing data / edge cases are handled

Goal¶

Create a modeling-ready dataset with one row per (grid cell, month) and a binary outcome:

Outcome $Y_{g,t}$: whether at least one notable fire occurs in grid cell $g$ in month $t+1$.

This “next month” target avoids same-month leakage and matches the project objective of near-future prediction.

Load Data¶

Key variable definitions (USDA Forest Service metadata):

  • FIRE_YEAR: Calendar year the fire was discovered
  • DISCOVERY_DOY: Day-of-year the fire was discovered
  • FIRE_SIZE: Final fire size (acres)
  • LATITUDE, LONGITUDE: Ignition location (decimal degrees)
  • STATE: State in which the fire burned

For reduce extreme sparsity, the panel should limit the grid cells with at least one notable fire record during the time. So, rather than compare across all grid cells, the prediction work should be the occurrence of wildfires in historically fire-active grid cells over the next month.

Next, we construct the complete (grid, month) panel. We include both positive (fire) and negative (no-fire) observations for:

  • All grid cells that have ever recorded at least one notable fire in grid_month_fire, and
  • All months within the observed study period., i.e.,the range of ym.

This helps create a completed panel with the selected grids, because the predictions are highly imbalanced:grid-month observations is 0.89% have y_next = 1, whereas 146,265 observations is 99.11% have y_next = 0. It shows that the notable fire events seldom happen.

Prediction target¶

For each (grid,month), define the indicator for the fire in the next month, i.e., $Y_{g,t}$:

  • y_next = 1 if a notable fire occurs in that grid cell in the next month

  group_by(lat_bin, lon_bin, ym) %>%
  summarise(n_fires = n(), mean_fire_size = mean(FIRE_SIZE), .groups = "drop")

3. Modeling¶

We fit at least two models:

  1. Baseline: Logistic regression (interpretable benchmark)
  2. Model of choice: Gradient boosted trees (XGBoost)
In [16]:
model_df <- panel %>%
  mutate(y_next = factor(y_next, levels = c(0, 1))) %>%
  select(y_next, lat_bin, lon_bin, state, 
         month_cos, month_sin, year, 
         n_fires, fire_lag12, fire_roll3, fire_roll12, size_roll3, size_roll12)

There are 147576 grid-month observations in the final modeling dataset. The binary outcome is y_next, and the predictors include spatial variables:lat_bin, lon_bin, state , seasonal variables: month_cos, month_sin, year, and lagged fire-history variables: n_fires, fire_lag12, fire_roll3, fire_roll12, size_roll3, and size_roll12. These predictors can help capture geographic variation, seasonality, and short- to long-term wildfire persistence.

Logistic regression is used as the baseline model because it provides an interpretable benchmark for a binary prediction problem. The baseline model can be written as

$$ \text{logit}(p_{g,t}) = \beta_0 + \text{spatial terms} + \text{seasonal terms} + \text{fire-history terms}. $$

This baseline model is useful because it provides a clear metric for comparing models and shows how each prediction relates to the logarithmic probability that a notable fire will occur in the next month.

As a second example, we use XGBoost, a gradient-boosted decision tree technique that works particularly well with tabular data. Since the risk of wildfire can depend on nonlinear effects and interactions between geography, season, and recent burn history, XGBoost is a particularly interesting choice in this situation. For example, the effects of recent fires can vary by month or location, and a simple linear model would not be sufficient to capture these relationships. Consequently, XGBoost is a flexible alternative that more effectively identifies complex predictive patterns in the data.

XGBoost predicts notable wildfires next month by combining lots of decision trees, each one built on improves errors of the previous one.

4. Model Fitting and Validation¶

Validation respects the temporal structure via expanding window. For each fold:

  • Train on years $\le t$
  • Validate on year $t+1$ with $t\geq 3$

We report AUC for both models and include a diagnostic plot for AUC across time folds.

The expanding window approach is suitable for forecasting future wildfire risk because it preserves the temporal order of the data. In each fold, these models are trained on data from the all available data up (t) and tested on the next year (t+1), so as not to use any information about the future to predict future outcomes. This reduces temporal bias and more accurately reflects the real-world forecasting situation, where wildfire risk would be assessed only from the historical information available at the time.

In [17]:
years_sorted <- sort(unique(model_df$year))
min_train_years <- 3

set.seed(123)

feat_formula <- y_next ~ lat_bin * lon_bin + state + month_cos + month_sin + 
  year + n_fires + fire_lag12 + fire_roll3 + fire_roll12 + size_roll3 + size_roll12

fold_results <- tibble(
  train_end_year = years_sorted[min_train_years:(length(years_sorted) - 2)],
  valid_year = years_sorted[(min_train_years + 1):(length(years_sorted) - 1)]
) %>%
  mutate(
    auc_glm = map2_dbl(train_end_year, valid_year, function(tr_end, va_year) {
      train <- model_df %>% filter(year <= tr_end)
      valid <- model_df %>% filter(year == va_year)
      
      fit <- suppressWarnings(glm(feat_formula, data = train, family = binomial))
      p_hat <- predict(fit, newdata = valid, type = "response")
      
      as.numeric(pROC::auc(pROC::roc(response = valid$y_next, predictor = p_hat, quiet = TRUE)))
    }),
    
    auc_xgb = map2_dbl(train_end_year, valid_year, function(tr_end, va_year) {
      train <- model_df %>% filter(year <= tr_end)
      valid <- model_df %>% filter(year == va_year)
      
      new_data <- bind_rows(train, valid) %>%
        mutate(state = factor(state, levels = levels(model_df$state)))
      
      x_all <- model.matrix(feat_formula, data = new_data)[, -1, drop = FALSE]
      x_train <- x_all[seq_len(nrow(train)), , drop = FALSE]
      x_valid <- x_all[(nrow(train) + 1):nrow(new_data), , drop = FALSE]
      
      y_train <- as.numeric(train$y_next) - 1
      y_valid <- as.numeric(valid$y_next) - 1
      
      scale_pos_weight <- sum(y_train == 0) / sum(y_train == 1)
      
      dtrain <- xgb.DMatrix(data = x_train, label = y_train)
      dvalid <- xgb.DMatrix(data = x_valid, label = y_valid)
      
      xgb_model <- xgb.train(
        params = list(
          objective = "binary:logistic",
          eval_metric = "auc",
          booster = "gbtree",
          eta = 0.05,                   
          max_depth = 3,                
          min_child_weight = 5,         
          subsample = 0.7,              
          colsample_bytree = 0.7,       
          gamma = 1,                    
          lambda = 1,                   
          alpha = 1,                    
          scale_pos_weight = scale_pos_weight,
          nthread = 8                   
        ),
        data = dtrain,
        evals = list(train = dtrain, valid = dvalid),
        nrounds = 1000,                 
        early_stopping_rounds = 50,     
        verbose = 0                     
      )
      
      p_hat_xgb <- predict(xgb_model, dvalid)
      as.numeric(pROC::auc(pROC::roc(response = valid$y_next, predictor = p_hat_xgb, quiet = TRUE)))
    })
  )

Because the output are highly imbalance, AUC is used as the primary evaluation metric. There are rarely any notable fires in the next month compared to the non-fire observations. Because a model might achieve high accuracy by forecasting largely zeros, this indicates that the overall accuracy is misleading. Higher-risk grid-months are ranked higher than lower-risk ones across various classification criteria by the algorithm, which is more visibly represented by AUC.

XGBoost has continually exceeded logistic regression on AUC metrics in previous years, showing that non-linear effects and interactions increase predictive power beyond a linear baseline. Furthermore, the differences across folds indicate that some fire seasons will inherently be more complex by nature than others. This outcome is in line with the earlier EDA finding that fire risk varies over time and is not constant between years.

5. Summary¶

This project uses historical notable wildfire records to make a next-month prediction framework. For each grid cell and each month, it predicts whether there will be at least one significant wildfire in the next month. After filtering out notable fires, we assemble a complete panel of grid observations for a given month to predict the next month, as well as the history of seasonal and re-recorded wildfires. The probability of a positive result in the next month is only 0.89%, so the data are strongly imbalanced.

There are several consistent trends that occur from the exploratory analysis. First of all, the inclusion of regional variables is supported by the fact that next-month wildfire risk is not spatially uniform, with certain grid cells consistently appearing as higher-risk locations. Then, the probability of notable fires strongly depends on seasonality, month, and motivates the use of cyclical month features like month_sin and month_cos. After that, recent wildfire history has research value: In the last 12 months, grid-months with more wildfire activity have a higher probability in the next month. And also, risk is flexible across years, making wildfire prediction hard to predict consistently over time.

In the modeling part, we used an expanding-window validation strategy that maintains temporal order to compare XGBoost with a baseline logistic regression model. Both models indicate they have the ability to predict, but XGBoost performs better over validation folds. The mean validation AUC was 0.784 for XGBoost, which has an absolute improvement of 0.089 over 0.695 for logistic regression. It means that predictive skill can be improved beyond a linear baseline by nonlinear effects and interactions in location, seansonality, and recent fire history.

Also, nothing is perfect; this analysis has some important limitations. The dataset used for model building is not a complete historical record of wildfires. Rather, it is a sample of forest fire records, and the 1° × 1° grid is relatively coarse inaccurate for representing local forest fire conditions. Furthermore, the predictor factors are based solely on the forest fire records and neglect some potential external variables such as climate, temperature, vegetation, and human activities. To improve the accuracy of future predictions, we can include richer environmental correlates and use a better spatial resolution.
